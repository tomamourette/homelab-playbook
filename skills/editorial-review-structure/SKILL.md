# Editorial Review (Structure)

**Core Utility** | **Model:** `/model quality` (Groq) | **Cost:** ~$0.01

## When to Use

Review a document's structure â€” headings, sections, completeness, template compliance.

## Execution

Handle locally. Switch to `/model quality` for structured analysis.

1. Read the document
2. Review for:
   - **Completeness:** Are all expected sections present?
   - **Heading hierarchy:** Is the H1/H2/H3 structure logical?
   - **Cross-references:** Are links valid? Are related docs referenced?
   - **Template compliance:** Does it follow the project's doc templates?
   - **Missing sections:** What should be added?
3. Provide specific structural recommendations

## Output

A structural review with specific recommendations for section additions, reordering, or removal.
