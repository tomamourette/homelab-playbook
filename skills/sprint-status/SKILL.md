## When to Use

Check the status of the current sprint. Which stories are done, in progress, or blocked?

## Execution

Handle locally on Gemini Pro.

1. Read the current sprint plan
2. Check the status of each story:
   - Look for completed implementation (commits, code changes)
   - Check for test results
   - Identify blockers
3. Report:
   - **Completed:** Stories that are done
   - **In Progress:** Stories being worked on
   - **Blocked:** Stories that can't proceed and why
   - **Remaining:** Stories not yet started
   - **Sprint health:** On track / At risk / Behind

## Output

A status summary reported to the user. No file output needed unless the user requests it.
